# 企业微信智能机器人 — 回调和回复的加解密方案

> 原始文档：https://developer.work.weixin.qq.com/document/path/101033
> 最后更新：2025年

---

## 一、概述

企业微信智能机器人的回调消息格式为 **JSON**（注意：普通企业应用回调为 XML，智能机器人为 JSON）。

所有回调消息和被动回复消息都需要加密/解密。企业微信提供官方加解密库（支持 C++/Python/PHP/Java/C#）。

**关键特殊说明：** 企业内部智能机器人场景中，`receiveid` 传**空字符串** `""`（与普通应用回调不同）。

---

## 二、配置参数

| 参数 | 说明 |
|------|------|
| **Token** | 自定义字符串（英文/数字，不超过 32 位），用于生成消息签名 |
| **EncodingAESKey** | 消息加解密密钥的 Base64 编码，固定 43 个字符（从 a-z、A-Z、0-9 中选取） |
| **ReceiveId** | 企业内部智能机器人场景：**传空字符串 `""`** |

**EncodingAESKey 解码后为 32 字节的 AES 密钥。**

---

## 三、AES 加密规格

| 规格项 | 值 |
|--------|-----|
| 算法 | AES |
| 模式 | CBC |
| 填充方式 | PKCS#7（填充至 32 字节的倍数） |
| IV（初始向量） | 取 AESKey 的前 16 字节 |
| 密钥长度 | 32 字节（AESKey 解 Base64 后） |

## 四、URL 验证（GET 请求）

### 4.1 企业微信发送的 GET 请求格式

```
GET https://your-server.com/callback?msg_signature=ASDFQWEXZCVAQFASDFASDFSS&timestamp=13500001234&nonce=123412323&echostr=ENCRYPT_STR
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `msg_signature` | 消息签名，用于验证请求来自企业微信 |
| `timestamp` | 时间戳 |
| `nonce` | 随机数 |
| `echostr` | 加密后的随机字符串，需解密后原样返回 |

### 4.2 验证步骤

1. 对收到的参数值做 **URL decode** 处理
2. 将 `token`、`timestamp`、`nonce`、`echostr`（加密值）按**字典序排序**后拼接成字符串
3. 对拼接字符串进行 **SHA1** 哈希，编码为 16 进制小写字符串
4. 与 `msg_signature` 对比，相等则签名验证通过
5. 解密 `echostr` 得到明文 msg
6. 将明文 msg **原样返回**作为 HTTP 响应

> 可直接使用企业微信加解密库的 `VerifyURL` 函数一步完成步骤 2-5。

## 五、接收回调消息（POST 请求）

### 5.1 企业微信推送的加密 JSON 格式

```json
{
  "encrypt": "ENCRYPTED_MSG_CONTENT",
  "msgsignature": "MSG_SIGNATURE",
  "timestamp": "TIMESTAMP",
  "nonce": "NONCE"
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `encrypt` | 加密后的消息内容（Base64 编码） |
| `msgsignature` | 消息签名 |
| `timestamp` | 时间戳（URL 参数中也有，值相同） |
| `nonce` | 随机数 |

### 5.2 解密步骤

1. 验证 `msgsignature` 签名（同 URL 验证的签名方式）
2. 对 `encrypt` 字段进行 Base64 解码
3. 使用 AES-CBC 解密（密钥 = EncodingAESKey 解码后的 32 字节）
4. 去掉 PKCS#7 填充
5. 解析明文结构体（见下方）

### 5.3 明文消息结构体

解密后得到的明文结构如下：

```
rand_msg = random(16B) + msg_len(4B) + msg + receiveid
```

| 部分 | 长度 | 说明 |
|------|------|------|
| `random` | 16 字节 | 随机字符串，丢弃 |
| `msg_len` | 4 字节 | `msg` 的长度，**网络字节序（大端）** |
| `msg` | `msg_len` 字节 | 实际消息内容（JSON 字符串） |
| `receiveid` | 剩余字节 | 接收者 ID（智能机器人场景为空字符串） |

**Python 伪代码示例：**
```python
content = decrypted_bytes[16:]                    # 去掉前 16 字节随机数
msg_len = int.from_bytes(content[0:4], 'big')    # 取 msg 长度（大端序）
msg = content[4:4 + msg_len]                      # 提取 msg
receiveid = content[4 + msg_len:]                 # 剩余为 receiveid
msg_json = json.loads(msg)                         # 解析 JSON
```

## 六、回复消息时的加密

### 6.1 加密步骤

1. 构造明文 JSON 消息包（如流式消息、模板卡片等）
2. 生成 16 字节随机字符串
3. 构造明文结构体：`random(16B) + msg_len(4B) + msg + receiveid`（receiveid 为空字符串）
4. 进行 PKCS#7 填充（填充至 32 字节的倍数）
5. AES-CBC 加密
6. Base64 编码得到 `encrypt` 字段值
7. 生成签名（`token`、`timestamp`、`nonce`、`encrypt` 排序后 SHA1）
8. 构造密文响应包

### 6.2 密文响应包格式

```json
{
  "encrypt": "BASE64_ENCRYPTED_MSG",
  "msgsignature": "SHA1_SIGNATURE",
  "timestamp": "TIMESTAMP",
  "nonce": "NONCE"
}
```

## 七、签名计算方法

**适用于：** URL 验证签名、消息签名验证、回复消息签名生成

**计算步骤：**

1. 将以下四个值按**字典序（ASCII 升序）**排序：
   - `token`
   - `timestamp`
   - `nonce`
   - `encrypt`（加密后的消息内容，验证 URL 时用 `echostr`）
2. 拼接成一个字符串（无分隔符）
3. 对拼接字符串进行 **SHA1** 哈希
4. 编码为 **16 进制小写字符串**（40 个字符）

**Python 示例：**
```python
import hashlib

def compute_signature(token, timestamp, nonce, encrypt):
    items = sorted([token, timestamp, nonce, encrypt])
    s = ''.join(items)
    return hashlib.sha1(s.encode('utf-8')).hexdigest()
```

## 八、不同场景中 ReceiveId 的含义

| 场景 | ReceiveId 值 |
|------|-------------|
| 企业自建应用消息回调 | `corpid`（企业 ID） |
| 第三方应用回调 | `suiteid`（套件 ID） |
| **企业内部智能机器人** | **空字符串 `""`** |

## 九、官方加解密库使用示例（Java）

```java
// 初始化（智能机器人场景 corpid/receiveid 传空字符串）
WXBizMsgCrypt wxcpt = new WXBizMsgCrypt(
    token,            // 你配置的 Token
    encodingAESKey,   // 你配置的 EncodingAESKey
    ""                // receiveid：智能机器人传空字符串
);

// 验证 URL（GET 请求处理）
String echoStr = wxcpt.VerifyURL(
    msgSignature,   // URL 参数 msg_signature
    timestamp,      // URL 参数 timestamp
    nonce,          // URL 参数 nonce
    echostr         // URL 参数 echostr
);
// 将 echoStr 原样返回

// 解密消息（POST 请求处理）
String plainMsg = wxcpt.DecryptMsg(
    msgSignature,   // 请求体 msgsignature
    timestamp,      // 请求体 timestamp（或 URL 参数）
    nonce,          // 请求体 nonce
    encryptMsg      // 请求体 encrypt
);
// plainMsg 即为明文 JSON 字符串

// 加密回复消息
String encryptedReply = wxcpt.EncryptMsg(
    replyMsgJson,   // 明文 JSON 字符串
    timestamp,
    nonce
);
// encryptedReply 即为密文响应包 JSON 字符串
```

## 十、常见问题

**Q: 智能机器人回调格式与普通应用回调有何区别？**
A: 智能机器人回调是 **JSON** 格式，普通应用回调是 **XML** 格式。加解密逻辑相同，但消息解析方式不同。

**Q: receiveid 为什么传空字符串？**
A: 企业内部智能机器人没有明确的 receiveid（不同于企业应用用 corpid 或第三方用 suiteid），官方规定传空字符串。

**Q: 如何区分回调来自企业微信而非伪造？**
A: 通过 `msg_signature` / `msgsignature` 验证签名，签名包含 token，伪造者不知道 token 无法生成合法签名。

# 企业微信智能机器人 — 接收消息

> 原始文档：https://developer.work.weixin.qq.com/document/path/100719
> 最后更新：2025年

---

## 一、概述

当用户与智能机器人发生交互，向机器人发送消息时，交互事件将**加密**后回调给开发者设置的接收消息 URL。

**注意事项：**
- 接收消息是加密的，需按加解密方案解密后处理
- 用户与同一个智能机器人最多同时有 **3 条消息交互**中
- 开发者接收到消息后，可选择：
  - 生成流式消息回复（调用大模型/AI Agent）
  - 直接回复模板卡片消息

---

## 二、支持的消息类型

| 消息类型 | 场景 |
|---------|------|
| 文本消息（`text`） | 用户群里 @ 机器人 或单聊发送文本 |
| 图文混排消息（`mixed`） | 用户群里 @ 机器人 或单聊发送图文混合内容 |
| 图片消息（`image`） | 用户单聊发送图片（群聊不支持） |
| 语音消息（`voice`） | 用户单聊发送语音（群聊不支持） |
| 本地文件消息（`file`） | 用户单聊发送文件（群聊不支持） |

---

## 三、消息通用字段

所有消息类型共有以下字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `msgid` | string | 消息唯一 ID，用于消息排重 |
| `aibotid` | string | 智能机器人的 ID |
| `chatid` | string | 群聊 ID（仅 `chattype` 为 `group` 时存在） |
| `chattype` | string | 聊天类型：`group`（群聊）/ `single`（单聊） |
| `from` | object | 发消息的用户信息 |
| `from.userid` | string | 发消息的用户 ID |
| `from.corpid` | string | 用户所在企业 ID（部分场景包含） |
| `response_url` | string | 用于回复消息的 URL（1小时有效，只能用一次） |
| `msgtype` | string | 消息类型 |
| `quote` | object | 引用消息内容（有引用时附带） |

---

## 四、各消息类型 JSON 示例

### 4.1 文本消息（群聊 @ 机器人）

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "response_url": "https://qyapi.weixin.qq.com/cgi-bin/aibot/response?response_code=RESPONSE_CODE",
  "msgtype": "text",
  "text": {
    "content": "@RobotA hello robot"
  }
}
```

### 4.2 文本消息（带引用，群聊）

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "response_url": "RESPONSEURL",
  "msgtype": "text",
  "text": {
    "content": "@RobotA hello robot"
  },
  "quote": {
    "msgtype": "text",
    "text": {
      "content": "这是今日的测试情况"
    }
  }
}
```

**`quote` 字段说明：**

| 字段 | 说明 |
|------|------|
| `quote.msgtype` | 被引用消息的类型（`text` / `image` 等） |
| `quote.text.content` | 被引用的文本内容 |

---

### 4.3 图文混排消息（群聊 @ 机器人）

当用户发送图文混合类型消息时，`msgtype` 为 `mixed`，其中 `mixed.msg_item` 是一个数组，每个元素有独立的 `msgtype`：

```json
{
  "msgid": "CAIQrcjMjQYY/NGagIOAgAMg6PDc/w0=",
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "response_url": "RESPONSEURL",
  "msgtype": "mixed",
  "mixed": {
    "msg_item": [
      {
        "msgtype": "text",
        "text": {
          "content": "@机器人 这是今日的测试情况"
        }
      },
      {
        "msgtype": "image",
        "image": {
          "url": "https://ww-aibot-img-xxx.cos.ap-guangzhou.myqcloud.com/..."
        }
      }
    ]
  },
  "quote": {
    "msgtype": "text",
    "text": {
      "content": "这是今日的测试情况"
    }
  }
}
```

**`mixed.msg_item` 数组中各元素字段：**

| 字段 | 说明 |
|------|------|
| `msgtype` | 元素类型：`text` / `image` |
| `text.content` | 文本内容 |
| `image.url` | 图片 URL（临时链接，有效期较短） |

### 4.7 流式消息刷新推送（stream）

若开发者回复了流式消息（`stream` 类型），企业微信会不断向回调 URL 推送流式消息刷新事件：

```json
{
  "msgid": "CAIQz7/MjQYY/NGagIOAgAMgl8jK/gI=",
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "msgtype": "stream",
  "stream": {
    "id": "STREAMID"
  }
}
```

**说明：**
- 从用户发消息开始，最多等待 **6 分钟**，超过则结束推送
- `stream.id` 对应开发者回复流式消息时设置的 `stream.id`，用于关联

---

## 五、消息排重建议

| 排重方式 | 适用场景 |
|---------|---------|
| 使用 `msgid` 排重 | 有 `msgid` 的消息（推荐） |
| 使用 `FromUserName + CreateTime` 排重 | 事件类型消息 |

---

## 六、响应要求

- 企业微信服务器在 **5 秒内**收不到响应会断掉连接并重试，总共重试 3 次
- 建议接收到回调后立即应答（返回 200 空包），业务逻辑异步处理
- 若需要长时间处理，返回空包后使用 `response_url` 主动回复

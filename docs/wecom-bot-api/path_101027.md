# 企业微信智能机器人 — 接收事件

> 原始文档：https://developer.work.weixin.qq.com/document/path/101027
> 最后更新：2025/11/25

---

## 一、概述

当用户与智能机器人发生特定交互（非发消息），会触发事件回调。事件同样以加密格式推送到开发者回调 URL。

**重要限制：** 企业微信服务器在 **5 秒内**收不到响应会断掉连接，丢弃该回调事件（不重试，与消息不同）。

---

## 二、事件通用格式

所有事件共有以下结构：

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "create_time": 1700000000,
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "single",
  "from": {
    "corpid": "wpxxxx",
    "userid": "USERID"
  },
  "msgtype": "event",
  "event": {
    "eventtype": "eventtype_name",
    "eventtype_name": {}
  }
}
```

**通用字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `msgid` | string | 消息唯一 ID |
| `create_time` | int | 事件创建时间戳（Unix 时间戳） |
| `aibotid` | string | 智能机器人 ID |
| `chatid` | string | 群聊 ID（群聊事件时存在） |
| `chattype` | string | 聊天类型：`group`（群聊）/ `single`（单聊） |
| `from.corpid` | string | 用户所在企业 ID |
| `from.userid` | string | 触发事件的用户 ID |
| `msgtype` | string | 固定为 `"event"` |
| `event.eventtype` | string | 事件类型名称 |

## 三、进入会话事件（`enter_chat`）

**触发条件：** 用户**当天首次**进入智能机器人单聊会话时触发。

**规则：**
- 若开发者未回复消息，用户当天再次进入也**不再推送**进入会话事件
- 可回复一条文本消息或模板卡片消息作为欢迎语

**回调示例：**

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "create_time": 1700000000,
  "aibotid": "AIBOTID",
  "from": {
    "corpid": "wpxxxx",
    "userid": "USERID"
  },
  "msgtype": "event",
  "event": {
    "eventtype": "enter_chat"
  }
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `event.eventtype` | 固定为 `"enter_chat"` |
| 无 `chatid` | 进入会话事件只在单聊场景触发，无群聊 ID |

## 四、模板卡片事件

当用户点击模板卡片中的按钮时，触发卡片交互事件。

### 4.1 按钮交互卡片事件

用户点击**按钮交互型**模板卡片的按钮时触发。

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "create_time": 1700000000,
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "response_url": "RESPONSEURL",
  "msgtype": "event",
  "event": {
    "eventtype": "template_card_event",
    "template_card_event": {
      "card_type": "button_interaction",
      "response_key": "BUTTON_KEY",
      "task_id": "TASK_ID"
    }
  }
}
```

**字段说明：**

| 字段 | 说明 |
|------|------|
| `event.template_card_event.card_type` | 卡片类型，如 `button_interaction` |
| `event.template_card_event.response_key` | 被点击按钮的 key 值 |
| `event.template_card_event.task_id` | 卡片任务 ID |

### 4.2 投票选择卡片事件

用户点击**投票选择型**模板卡片的选项时触发。

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "create_time": 1700000000,
  "aibotid": "AIBOTID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "response_url": "RESPONSEURL",
  "msgtype": "event",
  "event": {
    "eventtype": "template_card_event",
    "template_card_event": {
      "card_type": "vote_interaction",
      "response_key": "VOTE_KEY",
      "task_id": "TASK_ID",
      "selected_items": ["OPTION_ID_1"]
    }
  }
}
```

### 4.3 多项选择卡片事件

用户提交**多项选择型**模板卡片时触发。

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "create_time": 1700000000,
  "aibotid": "AIBOTID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },

  "msgtype": "event",
  "event": {
    "eventtype": "template_card_event",
    "template_card_event": {
      "card_type": "multiple_interaction",
      "response_key": "SUBMIT_KEY",
      "task_id": "TASK_ID",

          "question_key": "QUESTION_KEY",
```

## 五、用户反馈事件（`feedback_event`）

开发者在回复流式消息时可设置反馈信息（`feedback.id`），当用户对消息点赞/点踩后，触发用户反馈事件。

**用途：** 用于复盘智能机器人的回复效果。

**限制：**
- 若智能机器人回复消息中**未设置反馈信息**，则用户点击反馈后**不会触发**反馈事件
- 该事件目前**仅支持回复空包**，不支持回复新的消息或更新卡片消息

**回调示例：**

```json
{
  "msgid": "CAIQ16HMjQYY/NGagIOAgAMgq4KM0AI=",
  "create_time": 1700000000,
  "aibotid": "AIBOTID",
  "chatid": "CHATID",
  "chattype": "group",
  "from": {
    "userid": "USERID"
  },
  "msgtype": "event",
  "event": {
    "eventtype": "feedback_event",
    "feedback_event": {
      "id": "FEEDBACKID",
      "type": 2,
      "content": "能再详细一些么",
      "inaccurate_reason_list": [2, 4]
    }
  }
}
```

**`feedback_event` 字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 反馈事件唯一 ID（对应回复消息中设置的 `feedback.id`） |
| `type` | int | 反馈类型：`1`=点赞，`2`=点踩 |
| `content` | string | 用户填写的反馈内容文本（可选） |
| `inaccurate_reason_list` | array(int) | 不准确原因列表（枚举值，点踩时可选填） |

---

## 六、事件类型汇总

| 事件类型（eventtype） | 触发场景 |
|---------------------|---------|
| `enter_chat` | 用户当天首次进入单聊会话 |
| `template_card_event` | 用户点击按钮交互/投票/多项选择卡片 |
| `template_card_menu_event` | 用户点击卡片右上角菜单 |
| `feedback_event` | 用户对机器人回复消息点赞/点踩 |

# 企业微信智能机器人 — 概述

> 原始文档：https://developer.work.weixin.qq.com/document/path/101039
> 最后更新：2025年

---

## 一、功能介绍

企业微信智能机器人是企业微信 2025 年推出的面向内部协同的智能机器人功能。与传统群机器人（Webhook）相比，智能机器人支持被内部群 @ 后获取消息并回复，支持接入 AI/大模型，并通过 API 模式实现复杂交互。

**主要特点：**
- 支持 AI/RAG 能力：接入大模型进行智能问答，结合知识库检索
- 交互模式：联系人搜索、群聊 @ 问答、私聊，支持拉入群聊
- 接入方式：API 模式（非 Webhook），需处理加密回调、Token 验证

---

## 二、API 模式说明

若智能机器人开启 **API 模式**，当用户与智能机器人交互时，企业微信会向机器人 API 设置中的 **URL 回调地址**推送相关消息与事件。

- 接收消息与被动回复消息都需**加密**，加密方式参考《回调和回复的加解密方案》
- 开发者可根据接收到的消息或事件，被动回复消息
- 支持流式消息回复（用于大模型实时输出）

## 三、创建智能机器人（API 模式）

**操作步骤：**

1. 进入企业微信管理后台 → 管理工具 → 智能机器人
2. 点击"创建机器人"
3. 填写基本信息：名称、简介、可见范围
4. 选择试用模型（快速模型或推理模型）
5. 可配置知识库（支持上传文档、网页等）
6. 可设置转人工配置
7. 拉至最下方，选择 **API 模式**
8. 填写三个关键参数：
   - **URL**：你的回调服务地址（HTTP 或 HTTPS）
   - **Token**：自定义字符串（英文/数字，不超过32位），用于计算签名
   - **EncodingAESKey**：消息加解密密钥（43位 Base64 字符）

> **注意**：创建过程中可能遇到"域名主体校验未通过"报错，需配置备案主体与当前企业主体相同或有关联关系的域名。

## 四、URL 验证流程

企业微信在保存回调配置时，会先验证 URL 服务是否具备解析推送消息的能力。

**验证方式：** 企业微信向 URL 发一条 **GET 请求**，携带签名及密文参数。

**GET 请求示例：**
```
GET https://your-server.com/callback?msg_signature=ASDFQWEXZCVAQFASDFASDFSS&timestamp=13500001234&nonce=123412323&echostr=ENCRYPT_STR
```

**参数说明：**

| 参数 | 说明 |
|------|------|
| `msg_signature` | 消息签名，用于验证请求合法性 |
| `timestamp` | 时间戳 |
| `nonce` | 随机数 |
| `echostr` | 加密后的随机字符串，需解密后原样返回 |

**验证步骤：**
1. 解析各参数值（需做 URLdecode 处理）
2. 根据 Token，结合 `timestamp`、`nonce`、`echostr` 重新计算签名，与 `msg_signature` 对比
3. 解密 `echostr` 参数，得到明文 msg 字段
4. 将明文 msg 原样返回作为响应

> 验证通过后，配置开始生效，企业微信将向该 URL 推送用户交互消息。

## 六、与传统群机器人的区别

| 对比项 | 智能机器人（API 模式） | 群机器人（Webhook） |
|--------|----------------------|-------------------|
| 接入方式 | API 回调，需处理加密 | 简单 Webhook POST |
| 消息接收 | 支持群 @ 和单聊 | 不接收消息 |
| 消息类型 | 文本、图片、语音、文件、图文混排 | 仅发送消息 |
| 流式消息 | 支持（适合 LLM） | 不支持 |
| 复杂度 | 较高（需加密/解密） | 低 |
| 适用场景 | AI 助手、知识库问答、复杂交互 | 简单通知推送 |

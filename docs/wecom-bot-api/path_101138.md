# 企业微信智能机器人 — 主动回复消息

> 原始文档：https://developer.work.weixin.qq.com/document/path/101138
> 最后更新：2025年

---

## 一、概述

**主动回复**是指使用回调消息中携带的 `response_url` 字段，向企业微信发起 HTTP POST 请求来异步回复消息。

与被动回复（直接在 HTTP 响应中返回）不同，主动回复可以在接收回调后任意时间内（1小时内）发起，适合需要较长时间处理的场景。

**主动回复的消息不需要加密**，直接以明文 JSON 格式 POST 到 `response_url` 即可。

---

## 二、触发场景（哪些回调含 `response_url`）

| 场景 | 说明 |
|------|------|
| 用户向智能机器人发送消息 | 所有消息类型的回调均携带 `response_url` |
| 用户点击模板卡片按钮 | 模板卡片事件回调携带 `response_url` |

---

## 三、`response_url` 使用规则

| 规则 | 说明 |
|------|------|
| 有效期 | **1 小时**（从回调触发时计算） |
| 使用次数 | **只能调用一次**，调用后立即失效 |
| 调用方式 | HTTP POST 请求 |
| 内容编码 | `content` 字段必须为 **UTF-8** 编码 |
| 无需加密 | 主动回复消息体为明文 JSON |

**`response_url` 格式示例：**
```
https://qyapi.weixin.qq.com/cgi-bin/aibot/response?response_code=RESPONSE_CODE
```

---

## 四、支持的消息类型

主动回复支持以下消息类型：

| 消息类型 | `msgtype` 值 |
|---------|-------------|
| 文本 | `text` |
| Markdown | `markdown` |
| 图片 | `image` |
| 图文 | `news` |

---

## 五、各消息类型格式

### 5.1 文本消息（`text`）

```json
{
  "msgtype": "text",
  "text": {
    "content": "hello world"
  }
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `text.content` | string | 文本内容，UTF-8 编码，最长 2048 字节 |

---

### 5.2 Markdown 消息（`markdown`）

```json
{
  "msgtype": "markdown",
  "markdown": {
    "content": "# 今日报告\n\n**天气**：广州，29度，多云\n\n> 降雨概率：60%\n\n- 建议携带雨伞\n- 注意防晒"
  }
}
```

**支持的 Markdown 语法：**

| 语法 | 示例 |
|------|------|
| 标题 | `# 一级标题`，`## 二级标题` |
| 加粗 | `**加粗**` |
| 斜体 | `*斜体*` |
| 无序列表 | `- 列表项` |
| 有序列表 | `1. 列表项` |
| 引用 | `> 引用内容` |
| 代码（行内） | `` `code` `` |
| 代码块 | ` ```代码块``` ` |
| 链接 | `[文字](URL)` |
| 分割线 | `---` |

---

### 5.3 图片消息（`image`）

图片需要转为 Base64 编码：

```json
{
  "msgtype": "image",
  "image": {
    "base64": "/9j/4AAQSkZJRgABAQAAAQABAAD...",
    "md5": "MD5_OF_IMAGE_BINARY"
  }
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `image.base64` | string | 图片内容 Base64 编码 |
| `image.md5` | string | 图片原始二进制数据的 MD5 值（用于完整性校验） |

---

### 5.4 图文消息（`news`）

```json
{
  "msgtype": "news",
  "news": {
    "articles": [
      {
        "title": "中秋节礼品领取",
        "description": "今年中秋节公司有豪礼相送",
        "url": "https://work.weixin.qq.com",
        "picurl": "http://res.mail.qq.com/node/ww/wwopenmng/images/independent/doc/test_pic_msg1.png"
      }
    ]
  }
}
```

**`articles` 数组中每个对象的字段：**

| 字段 | 说明 |
|------|------|
| `title` | 图文标题 |
| `description` | 图文描述 |
| `url` | 点击后跳转的链接 |
| `picurl` | 图文的封面图片 URL（支持 JPG、PNG） |

## 六、发送请求示例

### 使用 curl 发送文本消息

```bash
curl 'https://qyapi.weixin.qq.com/cgi-bin/aibot/response?response_code=RESPONSE_CODE' \
  -H 'Content-Type: application/json' \
  -d '{
    "msgtype": "text",
    "text": {
      "content": "hello world"
    }
  }'
```

### 使用 curl 发送 Markdown 消息

```bash
curl 'https://qyapi.weixin.qq.com/cgi-bin/aibot/response?response_code=RESPONSE_CODE' \
  -H 'Content-Type: application/json' \
  -d '{
    "msgtype": "markdown",
    "markdown": {
      "content": "# 查询结果\n\n**广州**今日天气：29度，多云"
    }
  }'
```

### Node.js 示例

```javascript
const https = require('https');

async function replyWithResponseUrl(responseUrl, message) {
  const body = JSON.stringify({
    msgtype: 'text',
    text: { content: message }
  });

  return new Promise((resolve, reject) => {
    const url = new URL(responseUrl);
    const options = {
      hostname: url.hostname,
      path: url.pathname + url.search,
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'Content-Length': Buffer.byteLength(body)
      }
    };

    const req = https.request(options, (res) => {
      let data = '';
      res.on('data', chunk => data += chunk);
      res.on('end', () => resolve(JSON.parse(data)));
    });

    req.on('error', reject);
    req.write(body);
    req.end();
  });
}
```

## 七、特殊行为说明

**群聊中的引用消息：**
- 由于主动回复可能与用户交互触发的回调间隔较长时间
- 群聊中智能机器人主动回复消息时，会自动**引用触发回调的用户消息**（或被点击的模板卡片消息）
- 模板卡片消息不支持引用，会默认生成一条空消息进行引用

**响应结果：**
- 调用成功：HTTP 200，响应体为 `{"errcode":0,"errmsg":"ok"}`
- 调用失败：返回对应错误码

## 八、常见使用模式

### 模式一：立即返回空包 + 异步主动回复

适用于处理时间较长的场景（如调用外部 API、大模型推理等）：

```
用户消息到来
    ↓
立即返回 HTTP 200（空包）
    ↓
保存 response_url（有效期1小时）
    ↓
异步处理业务逻辑（调用 LLM 等）
    ↓
使用 response_url 发送最终回复
```

### 模式二：被动回复 + 主动更新

适用于需要先给用户一个即时反馈，再提供详细结果的场景（但注意被动回复只能发一次）。

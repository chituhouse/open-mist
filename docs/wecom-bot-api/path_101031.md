# 企业微信智能机器人 — 被动回复消息

> 原始文档：https://developer.work.weixin.qq.com/document/path/101031
> 最后更新：2025年

---

## 一、概述

**被动回复**是指在接收到用户消息/事件回调时，**直接在 HTTP 响应体中返回回复消息**。

**适用场景（仅以下两种情况支持被动回复）：**
1. 进入会话回调事件（`enter_chat`）
2. 接收消息回调（用户发消息）

**重要规则：**
- 首次回复时**必须**返回 stream 的 `id`（即便先回复模板卡片，也需要带上 stream id）
- 同一个消息**只能回复一次**
- 企业微信服务器 **5 秒内**收不到响应会断掉连接并重试（总共重试 3 次）
- 若无法在 5 秒内处理完毕，建议返回空包，改用**主动回复**（`response_url`）

## 二、流式消息回复（`stream`）

流式消息适用于大模型实时输出场景，可分多次推送文本内容。

### 2.1 基本结构

```json
{
  "msgtype": "stream",
  "stream": {
    "id": "STREAMID",
    "finish": false,
    "content": "**广州**今日天气：29度，大部分多云，降雨概率：60%",
    "msg_item": [
      {
        "msgtype": "image",
        "image": {
          "base64": "BASE64",
          "md5": "MD5"
        }
      }
    ],
    "feedback": {
      "id": "FEEDBACKID"
    }
  }
}
```

### 2.2 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `msgtype` | string | 是 | 固定为 `"stream"` |
| `stream.id` | string | 是 | 流式消息的唯一标识（开发者自定义，用于关联多次推送） |
| `stream.finish` | bool | 是 | 是否为最后一次回复：`false` 继续流式输出，`true` 结束 |
| `stream.content` | string | 是 | 本次回复的文本内容，支持 Markdown |
| `stream.msg_item` | array | 否 | 附加消息（如图片），**仅 `finish=true` 时支持** |
| `stream.feedback.id` | string | 否 | 反馈 ID，设置后用户可以对该消息点赞/点踩，触发反馈事件 |

### 2.3 content 格式说明

`stream.content` 支持常见 Markdown 格式：
- 标题：`# 一级标题`、`## 二级标题`
- 加粗：`**加粗文字**`
- 斜体：`*斜体文字*`
- 列表：`- 无序列表` / `1. 有序列表`
- 代码块：`` `code` `` 或 ` ```代码块``` `
- 链接：`[链接文字](URL)`

**特殊标签：**
- 若 `content` 包含 `<think></think>` 标签，客户端会将其中内容展示为"思考过程"（适用于推理模型）

**示例：**
```
<think>
用户询问广州天气，我需要查询实时数据...
</think>

**广州**今日天气：29度，大部分多云，降雨概率：60%
```

### 2.4 流式消息交互流程

```
用户发消息
    ↓
企业微信回调开发者 URL（POST，加密）
    ↓
开发者解密消息
    ↓
开发者调用大模型（流式输出）
    ↓
第1次响应：{"msgtype":"stream","stream":{"id":"xxx","finish":false,"content":"第一段文字"}}
    ↓
企业微信推送流式刷新回调（stream push）
    ↓
开发者收到流式刷新，继续回复
    ↓
...（多次推送）
    ↓
最后一次：{"msgtype":"stream","stream":{"id":"xxx","finish":true,"content":"最后一段文字"}}
```

**等待上限：** 从用户发消息开始，最多等待 **6 分钟**，超过则结束推送。

## 三、流式消息 + 模板卡片组合（`stream_with_template_card`）

可以同时回复流式文本和模板卡片：

```json
{
  "msgtype": "stream_with_template_card",
  "stream": {
    "id": "STREAMID",
    "finish": false,
    "content": "**广州**今日天气：29度，大部分多云，降雨概率：60%",
    "msg_item": [],
    "feedback": {
      "id": "FEEDBACKID"
    }
  },
  "template_card": {
    "feedback": {
      "id": "FEEDBACKID"
    }
  }
}
```

**规则：**
- 首次回复必须返回 stream 的 `id`
- `template_card` 可以首次回复，也可以在收到流式消息刷新事件时回复
- 同一个消息**只能回复一次**

---

## 四、仅回复模板卡片（`template_card`）

如果只需要回复模板卡片，不需要流式文本：

```json
{
  "msgtype": "template_card",
  "template_card": {
    "feedback": {
      "id": "FEEDBACKID"
    }
  }
}
```

完整的模板卡片格式见《模板卡片类型》文档（path/101032）。

## 五、回复消息的加密要求

被动回复消息**也需要加密**，与接收到的加密回调对称。

**加密步骤：**
1. 按明文协议构造 JSON 数据包（如上述示例）
2. 对明文消息进行 AES-CBC 加密，填充到 `encrypt` 字段
3. 计算签名
4. 回复密文 JSON 数据包

**密文数据包格式：**
```json
{
  "encrypt": "ENCRYPTED_MSG",
  "msgsignature": "MSG_SIGNATURE",
  "timestamp": "TIMESTAMP",
  "nonce": "NONCE"
}
```

详见《回调和回复的加解密方案》（path/101033）。
